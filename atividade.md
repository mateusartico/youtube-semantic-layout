# HTML-SEMANTICO

# Atividade: Domine as Tags Semânticas (HTML)

## Objetivo: O aluno deverá demonstrar conhecimento prático e teórico sobre as tags semânticas do HTML. Para cada tag solicitada, o aluno deve usar a tag em uma página HTML funcional e explicar (em comentários e em um arquivo README) o propósito da tag, por que a escolheu para aquele trecho e como isso ajuda em acessibilidade/SEO/manutenção.

### Instruções gerais

- Crie um projeto contendo ao menos os arquivos: index.html, styles.css, README.md e uma pasta assets/ com imagens.

- A página deve simular um site pequeno (por exemplo: restaurante, blog, portfolio, loja) com conteúdo suficiente para justificar o uso de todas as tags listadas abaixo.

- Para cada tag semântica exigida você deve:

- Inserir a tag no local adequado do HTML.

- Escrever um comentário HTML imediatamente acima da tag explicando: a função da tag, por que a usou naquele contexto e um exemplo de impacto em acessibilidade ou SEO.

- No README.md colocar uma explicação mais detalhada (2–4 frases) sobre cada tag usada.

- O código deve estar bem indentado e organizado; evite div desnecessárias quando existir tag semântica adequada.

## Lista obrigatória de tags semânticas (todas devem aparecer pelo menos 1 vez)

- "!DOCTYPE html" e "html lang="pt-br"" ok

"header" ok

"nav" (com uma lista de links) ok

"main" ok

"section" ok

"article" ok

"aside" ok

"footer" ok

"h1" a "h3" (uso hierárquico correto)

"p", "strong", "em" ok

"figure" e "figcaption" (com uma imagem em assets/)

"img" com alt adequado

"time" (com atributo datetime)

"address"

"table", "caption", "thead", "tbody", "tfoot" (tabela simples de dados relacionados)

"details" e "summary"

"mark"

"abbr" (com title)

"form" com "fieldset" e "legend" (pelo menos um campo input e um button)

"dialog" (uso demonstrativo; pode ser aberto via botão)

## Bônus técnico: usar atributos ARIA relevantes (role, aria-label, aria-describedby) em ao menos 3 elementos para reforçar acessibilidade — explique cada uso no README.

### Entregáveis

- index.html funcional que ilustre um site pequeno com as tags acima.

- styles.css com estilos mínimos (não precisa ser complexo, mas organize visualmente os elementos).

- README.md contendo explicações: breve descrição do projeto, por que cada tag foi usada e justificativas sobre acessibilidade/SEO.

- Comentários no código HTML explicando o uso de cada tag (como descrito nas instruções).

- Arquivos de mídia na pasta assets/ (pelo menos 1 imagem usada em "figure").
